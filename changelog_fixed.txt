2. FIX API 500 & 400 (CANDIDATE RESULTS & CERTIFICATE GENERATION)
   - BUGFIX: Memperbaiki error 500 "Cannot read properties of null (reading 'uid')" di `getCandidates`.
     Penyebab: Adanya data `userresults` yatim (orphaned) yang tidak memiliki user di tabel `users`.
     Solusi: Menambahkan pemeriksaan null defensif pada `resultController.js` untuk melewati data yang tidak valid (skip orphans).
   - BUGFIX: Memperbaiki error 400 "userResultId wajib diisi" saat generate sertifikat.
     Penyebab: Frontend mengirimkan ID dengan prefix `res-` (contoh: "res-44") sehingga `parseInt` di backend menghasilkan `NaN`.
     Solusi: Memperbarui `generate.vue` di frontend untuk membersihkan prefix `res-` sebelum mengirimkan ID ke API.
   - OPTIMASI: Menangani nilai rata-rata (average score) yang bernilai `null` agar tidak menyebabkan crash `NaN.toFixed(2)`.
   - BUGFIX: Memperbaiki error 500 "require is not defined" saat generate sertifikat.
     Penyebab: Library `nexaplot` menggunakan format ES Module (ESM) yang menyebabkan konflik `require` di Node.js 22.
     Solusi: Migrasi pemanggilan library di `certificateService.js` menggunakan dynamic `import()`.
   - VERIFIKASI: Telah diuji dengan script diagnostik `test_fix_500.js` dan `test_cert_gen.js`.
